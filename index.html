Different between LocalStorage vs SessionStorage:

LocalStorage:
  1. In HTML 5, there is a type of web storage object that allows saving data without worrying about it expiring.
  2. Designed for storage that spans multiple windows and it lasts beyond the current session
  3. 5MB/10MB storage
  4. Local storage is used for storing date locally on a user's browser, like saving user preference or others setting.
  5. It is persisted Through page reloads and stays with the browsers until it is cleared manually or Through a setting.
SessionStorage:
  1. Type of web storage object available in HTML 5 that allow storing data for one session
  2. Allows the user to carry out a single transaction but could be carrying out multiple transactions in different windows at the same time
  3. 5MB storage
  4. Session storage acts in a similar way to local Storage
  5. The Only difference here is that while data stored in local storage has an expiration date, the data stored in session storage will expire as soon as the page's session ends, that's it, when the page is closed

How does js work:

JavaScript is a programming language that runs on the client side and is one of the most efficient and widely used scripting languages. The phrase "client-side scripting language" refers to the fact that it operates on the client-side or on the client machine of web browsers, however it's worth noting that the client's web browser must also support JavaScript or be JavaScript enabled. Most current web browsers now accept JavaScript and even have their own JavaScript engines. Google Chrome, for example, has its own JavaScript engine called V8. It's crucial to understand how JavaScript works before you start creating it. The Document Object Model and how the web browser works are the two most crucial things to understand (DOM).A web browser loads a page, parses the HTML, and generates a Document Object Model (DOM) from the contents. The DOM gives your JavaScript code a live view of the web page. The browser will then acquire any pictures or CSS files that are related to the HTML. The CSS parser provides the CSS information.The DOM assembles the HTML and CSS to produce the web page initially. The JavaScript engine in the browser then loads JavaScript files and inline code, but does not run it right away. It waits for the HTML and CSS to load completely. After that, the JavaScript is run in the order in which it was written. As a result, JavaScript code updates the DOM, which is then displayed by the browser. It's crucial to keep everything in this sequence. The DOM elements could not be changed if the JavaScript did not wait for the HTML and CSS to finish.
